<p>&lt;div class=&quot;sl-w-full sl-mx-auto sl-px-20 sl-pt-20 sl-pb-8&quot; data-test=&quot;project-page&quot; style=&quot;max-width: 1500px;&quot;&gt;&lt;div class=&quot;sl-elements-article&quot;&gt;&lt;div class=&quot;sl-flex sl-relative&quot;&gt;&lt;div class=&quot;sl-prose sl-markdown-viewer sl-elements-article-content sl-overflow-x-auto sl-overflow-y-auto sl-flex-1 sl-p-2&quot;&gt;&lt;h1 class=&quot;sl-text-5xl sl-leading-tight sl-font-prose sl-font-bold sl-text-heading&quot; id=&quot;authentication&quot;&gt;Authentication&lt;/h1&gt;
&lt;hr/&gt;
&lt;p&gt;Hospitable protects our API endpoints using Personal Access Tokens, and &lt;a href=&quot;https://tools.ietf.org/html/rfc6749&quot; rel=&quot;noreferrer&quot; target=&quot;_blank&quot;&gt;OAuth 2.0&lt;/a&gt;'s Authorization flow grants. Depending on your application use case, it might depend which authentication grant you choose to go with.&lt;/p&gt;
&lt;h3 class=&quot;sl-link-heading sl-text-2xl sl-leading-snug sl-font-prose sl-font-semibold sl-text-heading&quot; id=&quot;what-authentication-should-i-use&quot;&gt;&lt;a class=&quot;sl-link sl-link-heading__link sl-inline-flex sl-items-center sl-text-current&quot; href=&quot;#what-authentication-should-i-use&quot;&gt;&lt;div&gt;What Authentication should I use?&lt;/div&gt;&lt;div class=&quot;sl-link-heading__icon sl-text-base sl-ml-4 sl-text-muted&quot;&gt;&lt;i aria-hidden=&quot;true&quot; class=&quot;sl-icon fal fa-link&quot; role=&quot;img&quot;&gt;&lt;/i&gt;&lt;/div&gt;&lt;/a&gt;&lt;/h3&gt;
&lt;p&gt;There are 2 forms of authentication to use the Hospitable API. Here's a short rundown on when you'd use either:&lt;/p&gt;
&lt;ul&gt;
&lt;li&gt;&lt;a href=&quot;#oauth-authorization-for-vendors&quot;&gt;OAuth Authorization flow for Vendors.&lt;/a&gt;
&lt;ul&gt;
&lt;li&gt;Like when logging into a website with Google or Twitter, Hospitable has it's own OAuth2 Authorization flow. This allows any third-party to integrate directly with Hospitable, without requiring the user to manually share credentials insecurely. This has the added bonus of being extra secure for both Hospitable users, and developers wishing to build on our platform.&lt;/li&gt;
&lt;/ul&gt;
&lt;/li&gt;
&lt;li&gt;&lt;a href=&quot;#personal-access-tokens&quot;&gt;Personal Access Tokens (PATs) for Personal API use.&lt;/a&gt;
&lt;ul&gt;
&lt;li&gt;As the name suggests, typically used for Personal API access. These tokens are scoped to your Hospitable account only, and should not be shared with any third-parties. You should only use these tokens for building and testing &lt;strong&gt;your own&lt;/strong&gt; Hospitable integration.&lt;/li&gt;
&lt;/ul&gt;
&lt;/li&gt;
&lt;/ul&gt;
&lt;p&gt;If you require any further information on which grant better suits your use case, &lt;a href=&quot;mailto:support@hospitable.com&quot;&gt;get in touch with support&lt;/a&gt;.&lt;/p&gt;
&lt;h2 class=&quot;sl-link-heading sl-text-4xl sl-leading-tight sl-font-prose sl-font-bold sl-text-heading&quot; id=&quot;oauth-authorization-for-vendors&quot;&gt;&lt;a class=&quot;sl-link sl-link-heading__link sl-inline-flex sl-items-center sl-text-current&quot; href=&quot;#oauth-authorization-for-vendors&quot;&gt;&lt;div&gt;OAuth Authorization for Vendors&lt;/div&gt;&lt;div class=&quot;sl-link-heading__icon sl-text-base sl-ml-4 sl-text-muted&quot;&gt;&lt;i aria-hidden=&quot;true&quot; class=&quot;sl-icon fal fa-link&quot; role=&quot;img&quot;&gt;&lt;/i&gt;&lt;/div&gt;&lt;/a&gt;&lt;/h2&gt;
&lt;p&gt;If you intend on building a Hospitable integration for others, you'll need to use our OAuth2 Authorization code flow. This follows the OAuth2 standard, which you can &lt;a href=&quot;https://www.oauth.com/oauth2-servers/access-tokens/authorization-code-request/&quot; rel=&quot;noreferrer&quot; target=&quot;_blank&quot;&gt;learn more about here&lt;/a&gt;.&lt;/p&gt;

&lt;p&gt;To get started, you'll need to request your own Vendor client credentials &lt;a href=&quot;https://form.typeform.com/to/n8bZPJIm&quot; rel=&quot;noreferrer&quot; target=&quot;_blank&quot;&gt;via this form&lt;/a&gt;. You'll be required to provide a:&lt;/p&gt;

&lt;p&gt;You can typically expect a response to your application within a few days. We may reach out for more information.&lt;/p&gt;

&lt;h3 class=&quot;sl-link-heading sl-text-2xl sl-leading-snug sl-font-prose sl-font-semibold sl-text-heading&quot; id=&quot;two-ways-to-initiate-the-connection-with-your-app&quot;&gt;&lt;a class=&quot;sl-link sl-link-heading__link sl-inline-flex sl-items-center sl-text-current&quot; href=&quot;#two-ways-to-initiate-the-connection-with-your-app&quot;&gt;&lt;div&gt;Two ways to initiate the connection with your app&lt;/div&gt;&lt;div class=&quot;sl-link-heading__icon sl-text-base sl-ml-4 sl-text-muted&quot;&gt;&lt;i aria-hidden=&quot;true&quot; class=&quot;sl-icon fal fa-link&quot; role=&quot;img&quot;&gt;&lt;/i&gt;&lt;/div&gt;&lt;/a&gt;&lt;/h3&gt;
&lt;p&gt;There are 2 ways a user can initiate the connection:&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;One-click OAuth initiated in Hospitable by a logged-in user&lt;/strong&gt;&lt;/p&gt;
&lt;p&gt;Once you have enabled OAuth, we will publish a dedicated page for your product in our &lt;a href=&quot;https://my.hospitable.com/apps&quot; rel=&quot;noreferrer&quot; target=&quot;_blank&quot;&gt;Apps marketplace&lt;/a&gt;. A logged in user will be able to initiate the connection through a Get Started button on this page.&lt;/p&gt;
&lt;p&gt;This redirects them directly to your configured redirect URL with an authorization &lt;code class=&quot;sl-font-mono sl-font-medium sl-mx-0.5 sl-px-1 sl-py-0.5 sl-bg-code sl-text-on-code sl-rounded sl-border&quot; style=&quot;font-size: 0.8125em;&quot;&gt;code&lt;/code&gt; added as a parameter to the URL. This eliminates the step where they have to be brought to a new page where they click to authorize the connection.&lt;/p&gt;
&lt;p&gt;&lt;strong&gt;Standard OAuth flow initiated by your customer from your app&lt;/strong&gt;&lt;/p&gt;
&lt;p&gt;This is initiated in your app. You must bring the user to the Hospitable Authorization page, where they will be asked to login and then click to authorize the connection.&lt;/p&gt;

&lt;h3 class=&quot;sl-link-heading sl-text-2xl sl-leading-snug sl-font-prose sl-font-semibold sl-text-heading&quot; id=&quot;1-setting-your-redirect-url&quot;&gt;&lt;a class=&quot;sl-link sl-link-heading__link sl-inline-flex sl-items-center sl-text-current&quot; href=&quot;#1-setting-your-redirect-url&quot;&gt;&lt;div&gt;1. Setting your redirect URL&lt;/div&gt;&lt;div class=&quot;sl-link-heading__icon sl-text-base sl-ml-4 sl-text-muted&quot;&gt;&lt;i aria-hidden=&quot;true&quot; class=&quot;sl-icon fal fa-link&quot; role=&quot;img&quot;&gt;&lt;/i&gt;&lt;/div&gt;&lt;/a&gt;&lt;/h3&gt;
&lt;p&gt;When you create your client credentials in the Partner Portal, you will be able to configure the redirect URL, where we will send your customers after they have authorized the connection between your app and their Hospitable account.&lt;/p&gt;
&lt;p&gt;By default, this URL is used for both methods described above. If you wish to set a different URL for the one-click connection, let our team know and we can configure that for you.&lt;/p&gt;
&lt;p&gt;You also have the option to set a custom URL for the one-click connection. We will not pass the authorization code, which means the user will simply be brought to your site, and you will need to navigate them at some point to the Hospitable Authorization page, following the standard flow. The custom URL supports state so you can associate the request with the user.&lt;/p&gt;
